Transformation: Node Reference to Node/Container Mapping

Migration relation type: Transformation
Migration scope: GMF Workbench => Sirius Workbench

Applicability conditions:

  • REQUIRED on Node Reference, Node Mapping, Compartment Mapping: Apply when the GMF Node Reference can be resolved to a referenced or owned Node Mapping; choose Siri us Container Mapping when the referenced structure contains compartments or child containment, other wise choose Sirius Node Mapping.

Transformation intent

Source feature: Node Reference
Target feature(s): Node Mapping, Container Mapping
Automation: FullAutomation / COMPLEX
Relation: GMF Node References act as indirection points that bind semantic containment to graphical realizatio n via Node Mappings. Since Sirius distinguishes explicitly between simple nodes and containers, the migration must inspect the structural context of the referenced Node Mapping. By conditionally creat ing a Container Mapping when compartments exist, or a Node Mapping otherwise, the mapping preserves both the containment semantics and the visual hierarchy encoded in GMF diagrams.

Realization strategy


1. [A] take the node mapping (A) of the top node reference (B) or child reference (C)



2. [A] check if A has a compartment and a child reference (X)



3. [A] if X is true

    3.1. [A] create a container mapping under the default layer (for B) or under the parent reference's

 equivalent (for C)



    3.2. [A] take all the compartment mappings (G) of A



    3.3. [A] take all the visual representation compartments (H) of G



    3.4. [A] take all the child accesses (I) using the figure and accessor properties of H



    3.5. [A] take all the shapes (J) using the figure property of I



    3.6. [A] take all the LayoutData features properties of J



    3.7. [A] execute 'Computing Children Presentation for Container Mapping from Top Node/Child Referen

ce'





4. [A] else check if B/C has a sibling compartment (Y)



5. [A] if Y is true

    5.1. [A] take all the sibling compartment mappings (D)



    5.2. [A] take the compartment mapping in D that include B/C in the children property (E)



    5.3. [A] take E's equivalent container mapping (F)



    5.4. [A] create a (sub) node mapping under F
