Transformation: Multiple Label Mappings to Label

Migration relation type: Transformation
Migration scope: GMF Workbench => Sirius Workbench

Applicability conditions:

  • REQUIRED on Feature Label Mapping: Apply when one Sirius graphical element would need to display one or more GMF Feature Label Mappings as a single style-level label expression.

Transformation intent

Source feature: Feature Label Mapping
Target feature(s): Label Expression
Automation: NoDirectMapping / COMPLEX, MANUAL
Relation: GMF Feature Label Mappings and Sirius Label Expressions both serve the same fundamental purpose: def ining how semantic attributes are transformed into textual labels on diagram elements. However, GMF supports multiple independent label mappings per node, while Sirius enforces a single label expressi on per graphical element. This structural mismatch necessitates an approximation rather than a direc t mapping.

Operators: Mr. X

No-direct-mapping rationale: Because Sirius supports only one style - and consequently one label expression - per diagram element , all GMF Feature Label Mappings associated with the same node are merged into a single AQL expressi on. Line breaks are used to preserve label separation and readability.

Possible approximation feature(s): Label Expression
Approximation profile: COMPLEX, MANUAL
