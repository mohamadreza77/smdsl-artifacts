Transformation: Java 8/11 runtime to Java 17 baseline

Migration relation type: Transformation
Migration scope: Spring Boot 2.7.x / late Spring Boot 2 application => Spring Boot 3.0 application / migration target

Applicability conditions: None specified.

Transformation intent

Source feature: Java 8 or Java 11 runtime baseline
Target feature(s): Java 17 runtime and toolchain baseline, Java 17 language and runtime compatibility
Automation: OperatorHelp / MIX
Relation: Spring Boot 3 requires Java 17 or later. This is a platform baseline migration rather than a simple library rename because CI, IDE, compiler source/target settings, container images, and runtime envir onments must all agree.

Operators: Application Developer 2

Realization strategy


1. [A] update Maven or Gradle toolchain/source/target settings to Java 17



2. [A] update CI and container images to a Java 17 runtime



3. [M] inspect reflective-access, illegal-access, or removed-JDK-module issues



4. [M] run tests under Java 17 before upgrading Boot
