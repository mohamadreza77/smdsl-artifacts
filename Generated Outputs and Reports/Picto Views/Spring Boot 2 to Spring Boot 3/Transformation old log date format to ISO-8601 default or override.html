Transformation: old log date format to ISO-8601 default or override

Migration relation type: Transformation
Migration scope: Spring Boot 2.7.x / late Spring Boot 2 application => Spring Boot 3.0 application / migration target

Applicability conditions: None specified.

Transformation intent

Source feature: Spring Boot 2 log date format
Target feature(s): ISO-8601 log date/time format
Automation: OperatorHelp / MIX
Relation: Spring Boot 3 changes the default Logback/Log4j2 date format to ISO-8601 style. Applications dependi ng on log parsing may either accept the new format or restore the old pattern.

Operators: Application Developer 2

Realization strategy


1. [M] check whether log aggregation/parsing expects the old date format



2. [NONE] if old format must be preserved

    2.1. [A] set LOG_DATEFORMAT_PATTERN or logging.pattern.dateformat to the old pattern





3. [M] validate log parser behavior
