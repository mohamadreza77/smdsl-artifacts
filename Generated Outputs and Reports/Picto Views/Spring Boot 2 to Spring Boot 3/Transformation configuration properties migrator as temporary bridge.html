Transformation: configuration properties migrator as temporary bridge

Migration relation type: Transformation
Migration scope: Spring Boot 2.7.x / late Spring Boot 2 application => Spring Boot 3.0 application / migration target

Applicability conditions: None specified.

Transformation intent

Source feature: Legacy renamed or removed configuration properties
Target feature(s): Spring Boot properties migrator temporary compatibility bridge, Updated configuration properties, Removed properties migrator after migration
Automation: OperatorHelp / MIX
Relation: The properties migrator is a temporary compatibility bridge: it reports renamed/removed properties a nd can temporarily migrate them at runtime, but it must be removed after the configuration has been updated.

Operators: Application Developer 2

Realization strategy


1. [A] add spring-boot-properties-migrator as a runtime-only dependency



2. [A] start the application and collect diagnostics for renamed/removed properties



3. [M] edit application.properties or application.yml to use the new keys



4. [A] restart and confirm no migrator diagnostics remain



5. [A] remove the migrator dependency after migration
