Transformation: javax-based third-party libraries to Jakarta variants

Migration relation type: Transformation
Migration scope: Spring Boot 2.7.x / late Spring Boot 2 application => Spring Boot 3.0 application / migration target

Applicability conditions: None specified.

Transformation intent

Source feature: Java EE based third-party library variants
Target feature(s): Jakarta-compatible third-party library variants
Automation: OperatorHelp / MIX
Relation: Some libraries embed Java EE APIs in their public types. These cannot be fixed only by import rewrit ing; the library itself must have a Jakarta-compatible artifact or version.

Operators: Application Developer 2

Realization strategy


1. [A] detect third-party libraries exposing javax types



2. [M] find Jakarta-compatible version or artifact



3. [NONE] if no compatible artifact exists

    3.1. [M] replace the library or isolate it behind an adapter





4. [A] update dependency declaration and compile
