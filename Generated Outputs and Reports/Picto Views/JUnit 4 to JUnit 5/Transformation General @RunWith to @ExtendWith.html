Transformation: General @RunWith to @ExtendWith

Migration relation type: Transformation
Migration scope: JUnit 4 => JUnit 5 / JUnit Jupiter and JUnit Platform

Applicability conditions:

  • OPTIONAL on @RunWith, @ExtendWith: Applies only when the runner corresponds to an extension-supported integration such as Spring or Moc kito; obsolete JUnit4 runners may simply be removed.

Transformation intent

Source feature: @RunWith
Target feature(s): @ExtendWith
Automation: OperatorHelp / MIX
Relation: @RunWith no longer exists in Jupiter and is generally superseded by the Jupiter extension model. The mapping depends on the concrete runner.

Operators: Migration engineer / test developer

Realization strategy


1. [A] Inspect the runner class supplied to @RunWith.



2. [A] If the runner has an official Jupiter extension counterpart, replace @RunWith(R.class) wit

h @ExtendWith(E.class).



3. [M] If runner-specific annotations/configuration exist, migrate them according to that framewo

rk.



4. [M] If multiple runners were impossible in JUnit 4 but rules compensated, reconsider compositi

on using multiple extensions.



5. [M] If no extension counterpart exists, treat as custom runner redesign.
