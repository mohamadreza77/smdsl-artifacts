Transformation: Timeout Rule to @Timeout or Timeout Assertions

Migration relation type: Transformation
Migration scope: JUnit 4 => JUnit 5 / JUnit Jupiter and JUnit Platform

Applicability conditions: None specified.

Transformation intent

Source feature: Timeout Rule
Target feature(s): @Timeout, assertTimeout, assertTimeoutPreemptively
Automation: OperatorHelp / MIX
Relation: The JUnit 4 Timeout rule applies timeout behavior through the rule mechanism. Jupiter has annotation and assertion constructs instead of the rule.

Operators: Migration engineer / test developer

Realization strategy


1. [A] Detect @Rule Timeout or @ClassRule Timeout and its configured duration.



2. [A] If rule applies uniformly to each test, replace with class-level or method-level @Timeout.



3. [M] If timeout behavior is conditional or composed with other rules, inspect manually.



4. [M] If timeout is intended for only part of a test, use assertTimeout around that executable.



5. [M] Validate behavior under parallel execution and thread interruption policies.
