Transformation: @Ignore to @Disabled or Migration-Support IgnoreCondition

Migration relation type: Transformation
Migration scope: JUnit 4 => JUnit 5 / JUnit Jupiter and JUnit Platform

Applicability conditions:

  • OPTIONAL on @Ignore, @Disabled, IgnoreCondition, @EnableJUnit4MigrationSupport: Use @Disabled for full migration; use IgnoreCondition or @EnableJUnit4MigrationSupport only during g radual migration.

Transformation intent

Source feature: @Ignore
Target feature(s): @Disabled, IgnoreCondition, @EnableJUnit4MigrationSupport
Automation: OperatorHelp / MIX
Relation: @Ignore no longer exists in Jupiter. The normal target is @Disabled, but JUnit 4 @Ignore can also be supported temporarily via junit-jupiter-migrationsupport.

Operators: Migration engineer / test developer

Realization strategy


1. [A] If the class/method is being fully migrated to Jupiter, replace @Ignore with @Disabled and

 preserve the reason string when present.



2. [M] If a gradual migration keeps @Ignore temporarily on Jupiter tests, add junit-jupiter-migra

tionsupport and register IgnoreCondition or @EnableJUnit4MigrationSupport.



3. [M] If @Ignore is on a still-vintage JUnit 4 test, leave unchanged until that class is migrate

d.



4. [M] Verify build/test reports because ignored/disabled terminology may appear differently.
