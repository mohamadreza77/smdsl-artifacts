Transformation: @Test(expected) to assertThrows

Migration relation type: Transformation
Migration scope: JUnit 4 => JUnit 5 / JUnit Jupiter and JUnit Platform

Applicability conditions:

  • REQUIRED on @Test(expected), assertThrows: Applies when @Test declares expected = SomeException.class; target code must wrap the exact throwing statement or block.

Transformation intent

Source feature: @Test(expected)
Target feature(s): assertThrows, assertThrows block
Automation: OperatorHelp / MIX
Relation: JUnit 4 expected-exception tests declare only the exception type at annotation level. Jupiter assert Throws requires the operator or transformation to identify and wrap the specific executable statemen t or block expected to throw.

Operators: Migration engineer / test developer

Realization strategy


1. [A] Detect @Test(expected = E.class) and remove the expected attribute from @Test.



2. [M] Identify the minimal statement or expression that is expected to throw E. This information

 is often implicit in the source test body.



3. [M] Wrap that statement in assertThrows(E.class, () -> { ... }).



4. [M] If statements before the throwing line set up state, keep them before assertThrows.



5. [M] If statements after the throwing line existed in JUnit 4, mark them unreachable under the 

source semantics and inspect whether they should be removed or moved into target assertion

s.



6. [M] If the exception object is needed, assign the result of assertThrows to a variable and add

 message/cause assertions.



7. [A] Run the migrated test to ensure it fails if no exception is thrown and also fails for the 

wrong exception type.
